## 定位

2D 图像识别与卷积神经网络是「2D 计算机视觉」子域的基础，它负责把像素转化为可用于判断图像类别的特征喵。

掌握这一概念后，你将更容易理解如何在公路图像中定位裂缝与坑槽的「2D 目标检测」，以及如何精确描出病害轮廓的「图像分割与公路病害识别」喵。

## 类比

可以把卷积神经网络（CNN）想象成一支检查公路航拍图的分层巡检队伍，而一张数字图像就是铺在桌面上的巨大方格地图喵。

第一组巡检员拿着很小的透明检查模板，在地图上从左到右、从上到下滑动，每次只观察附近的一小块区域，这个模板就是卷积核，而它当前覆盖的区域就是局部感受野喵。

同一个巡检员不会因为走到图像左侧或右侧就更换判断标准，而是始终使用同一套模板寻找裂缝边缘，这种做法对应权值共享喵。

不同巡检员会携带不同模板，有的寻找明暗突变，有的寻找横向线条，有的寻找斜向纹理，因此一次卷积可以生成多张反映不同视觉线索的特征图喵。

第一层巡检员只汇报边缘和色彩变化，下一层把这些报告组合成粗糙纹理与裂缝片段，再下一层识别裂缝分叉、坑槽边界或车道线等较完整部件，最终的负责人据此判断整张图属于正常、裂缝还是坑槽喵。

池化就像把一份过于细密的巡检地图压缩成摘要，只保留每个小区域中最显著的证据，从而减少计算量并容忍病害位置发生少量偏移喵。

这套类比揭示了 CNN 的核心思想，即先用共享的小模板发现局部模式，再通过多层组合逐渐形成对完整物体或场景的理解喵。

## 核心机制

### 图像作为张量

计算机通常把彩色图像表示为三维张量，其中高度和宽度对应像素的空间位置，通道维度对应红、绿、蓝三种颜色喵。

在 PyTorch 中，一批图像通常采用 `[N, C, H, W]` 的排列方式，其中 `N` 是批量大小，`C` 是通道数，`H` 和 `W` 分别是图像高度与宽度喵。

例如，一批由 32 张 `224 × 224` RGB 图像组成的数据，其张量形状是 `[32, 3, 224, 224]`喵。

原始像素值常位于 `0～255`，训练前通常会先转换成浮点数并缩放，再按通道进行归一化，使不同输入的数值范围更稳定喵。

图像张量不仅包含颜色数值，还隐含着重要的空间结构，因为相邻像素往往来自同一条边缘、同一块纹理或同一个物体喵。

### 卷积操作与卷积核

卷积层的任务是让一个较小的卷积核在图像上滑动，并在每个位置计算局部像素与卷积核权重的加权和喵。

对于一个 `3 × 3` 卷积核，每次计算只观察九个相邻位置，因此它天然关注局部结构，而不是一开始就把整张图像的所有像素混在一起喵。

严格来说，深度学习框架中的 `Conv2d` 通常执行互相关运算，即卷积核在计算前不会翻转，但行业中仍习惯把它称为卷积喵。

卷积核的数值不是人工预先写死的规则，而是通过训练数据和反向传播自动学习得到的参数喵。

一个卷积层通常包含多个卷积核，每个卷积核产生一个输出通道，因此设置 `out_channels=32` 就表示该层会学习 32 种不同的局部特征检测方式喵。

对于 RGB 图像，一个卷积核的深度必须覆盖全部三个输入通道，所以一个空间尺寸为 `3 × 3` 的卷积核实际参数形状是 `[3, 3, 3]`喵。

卷积使用同一组权重扫描整张图像，这种权值共享显著减少了参数量，也使网络能够在不同位置识别相似的裂缝纹理喵。

如果输入整体向右移动几个像素，卷积生成的特征通常也会相应向右移动，这种性质称为平移等变性喵。

### Padding 与 Stride

如果卷积核只能完全落在图像内部，输出的高度和宽度会逐层缩小，而且图像边缘的像素被使用的次数也会少于中心像素喵。

填充（Padding）会在图像边缘补充额外像素，最常见的做法是补零，从而控制输出尺寸并保留更多边界信息喵。

对于步幅为 `1` 的 `3 × 3` 卷积，在四周填充 `1` 个像素就可以让输出的空间尺寸与输入相同，这种配置常被称为 same padding喵。

步幅（Stride）表示卷积核每次移动多少个像素，步幅为 `1` 时会逐格扫描，步幅为 `2` 时则会跳过部分位置并让输出尺寸大致减半喵。

输出尺寸可以用 `⌊(输入尺寸 + 2 × padding - 卷积核尺寸) / stride⌋ + 1` 计算，因此卷积核、填充和步幅共同决定特征图大小喵。

较大的步幅可以减少计算量并扩大后续神经元的有效感受野，但也可能过早丢失细小裂缝等高频信息喵。

### 激活函数与池化

如果多个卷积层之间没有非线性激活函数，那么无论堆叠多少层，它们整体仍然只能表示一次线性变换，无法学习复杂的视觉决策边界喵。

ReLU 激活函数使用 `max(0, x)` 把负值截断为零，同时保留正值，具有计算简单且有利于梯度传播的特点喵。

可以把 ReLU 理解为一道证据门槛，只有卷积核在当前位置发现足够匹配的模式时，对应信号才会继续传给后续层喵。

最大池化（Max Pooling）会在每个局部窗口中保留最大值，例如 `2 × 2` 最大池化会把四个相邻响应压缩成一个最强响应喵。

池化能够减少特征图尺寸、降低计算量，并让网络对小范围位置变化更稳定，但它也会损失精确空间信息喵。

对于需要精确描绘裂缝轮廓的图像分割任务，网络往往会谨慎下采样，并通过跳跃连接或上采样恢复细节喵。

### 层级特征与感受野

浅层卷积直接接触原始像素，因此更容易学习边缘方向、亮度变化和简单颜色对比等低级特征喵。

中间层会组合多个低级特征，从而识别沥青颗粒纹理、细长裂缝片段、坑槽边缘或规则车道线等结构喵。

深层会继续组合这些局部结构，形成对裂缝分叉、网状龟裂、完整坑槽乃至道路场景语义的表示喵。

虽然单个卷积核只观察局部窗口，但多层卷积堆叠后，深层神经元能够间接接收越来越大范围的输入信息，这个范围称为有效感受野喵。

这种从边缘到纹理、从纹理到部件、再从部件到物体的层级表示，是 CNN 处理视觉任务的重要能力喵。

需要注意的是，这些层级并不是人工规定的固定字典，而是网络在优化分类目标时逐渐形成的统计表示喵。

### 经典网络的演进

LeNet 展示了卷积、激活、池化和全连接层可以组成端到端图像识别系统，并成功用于手写数字识别等较简单任务喵。

AlexNet 在更大规模图像数据上使用更深的网络、ReLU、Dropout 和 GPU 训练，证明深度 CNN 能显著提升复杂自然图像分类性能喵。

VGG 进一步展示了重复堆叠小型 `3 × 3` 卷积的简洁设计原则，因为多个小卷积既能扩大感受野，也能插入更多非线性变换喵。

然而，网络变深后可能出现训练误差反而升高的退化问题，其原因不只是传统意义上的过拟合，还包括深层网络难以优化和梯度传播困难喵。

ResNet 引入残差连接，让一个模块学习残差函数 `F(x)`，并将输入直接相加得到 `y = F(x) + x`喵。

这条捷径为信息和梯度提供了更直接的传播通道，而且当额外层暂时学不到有用变换时，只要让 `F(x)` 接近零就能近似保留原输入喵。

残差连接使几十层乃至上百层网络更容易训练，也成为现代 CNN、检测器、分割网络以及部分 Transformer 架构中的基础设计喵。

### CNN 与 Transformer 的归纳偏置

CNN 在结构上预先假设视觉模式具有局部性，并假设同一种模式可能出现在图像的不同位置，因此使用局部连接和权值共享喵。

这些结构假设称为归纳偏置，它们让 CNN 即使在数据量相对有限时，也能较高效地学习边缘、纹理和局部形状喵。

视觉 Transformer（ViT）通常先把图像切分为 Patch，再把每个 Patch 视为 Token，并通过自注意力建立不同位置之间的关系喵。

ViT 的自注意力可以较早进行全局信息交互，但它不像卷积那样天然强调邻近像素和位置共享，因此通常需要位置编码以及更多数据或预训练来学习视觉结构喵。

在公路病害识别中，CNN 擅长捕捉局部裂缝纹理，而 Transformer 更容易直接建模相距较远区域之间的关系，实际系统也常把两者组合使用喵。

理解二者差异的重点不是判断谁绝对更好，而是识别 CNN 把局部性和平移等变性写进了网络结构，而 Transformer 更多地让模型从数据中学习关系喵。

### 公路病害分类如何落地

一个最基础的公路病害分类系统可以把摄像头图像或道路巡检图像裁成尺寸统一的图像块，并为每个图像块标注正常、裂缝或坑槽类别喵。

训练阶段把图像块送入 CNN，网络输出三个类别的原始分数，再通过交叉熵损失衡量预测与真实标签之间的差异喵。

反向传播会计算损失对每个卷积核参数的梯度，优化器随后更新参数，使正确类别的得分逐渐升高喵。

部署阶段对新图像执行同样的缩放和归一化，再选择得分最高的类别作为预测结果，同时保留概率或置信度用于告警阈值判断喵。

数据划分应以道路区段、采集批次或时间段为单位，避免同一路面的近重复图像同时进入训练集和验证集而造成虚假的高准确率喵。

类别准确率不能完整反映工程效果，尤其在病害样本稀少时，还应关注每类召回率、精确率、F1 值和混淆矩阵喵。

图像分类只能回答一张图主要属于什么类别，却不能直接指出病害位于哪里，也不能给出裂缝覆盖的精确像素范围喵。

如果任务需要在大图中框出多个坑槽，应继续学习「2D 目标检测」；如果任务需要计算裂缝长度、宽度或病害面积，则应继续学习「图像分割与公路病害识别」喵。

## 代码示例

下面的示例使用 PyTorch 定义并训练一个小型 CNN，对路面图像执行正常、裂缝和坑槽三分类喵。

数据目录需要按类别组织，`ImageFolder` 会根据子目录名称自动生成类别编号喵。

```text
road_data/
├─ train/
│  ├─ normal/
│  ├─ crack/
│  └─ pothole/
└─ val/
   ├─ normal/
   ├─ crack/
   └─ pothole/
```

```python
from pathlib import Path

import torch
from torch import nn
from torch.utils.data import DataLoader
from torchvision import datasets, transforms


# 固定随机种子，便于复现实验
torch.manual_seed(42)

DATA_DIR = Path("road_data")
BATCH_SIZE = 32
NUM_CLASSES = 3
EPOCHS = 10
LEARNING_RATE = 1e-3

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")


# 训练集使用轻量数据增强，验证集只做确定性预处理
train_transform = transforms.Compose(
    [
        transforms.RandomResizedCrop(128, scale=(0.8, 1.0)),
        transforms.RandomHorizontalFlip(),
        transforms.ColorJitter(brightness=0.2, contrast=0.2),
        transforms.ToTensor(),
        transforms.Normalize(
            mean=(0.485, 0.456, 0.406),
            std=(0.229, 0.224, 0.225),
        ),
    ]
)

val_transform = transforms.Compose(
    [
        transforms.Resize(144),
        transforms.CenterCrop(128),
        transforms.ToTensor(),
        transforms.Normalize(
            mean=(0.485, 0.456, 0.406),
            std=(0.229, 0.224, 0.225),
        ),
    ]
)


train_dataset = datasets.ImageFolder(
    DATA_DIR / "train",
    transform=train_transform,
)
val_dataset = datasets.ImageFolder(
    DATA_DIR / "val",
    transform=val_transform,
)

if train_dataset.class_to_idx != val_dataset.class_to_idx:
    raise ValueError("训练集与验证集的类别目录不一致")

if len(train_dataset.classes) != NUM_CLASSES:
    raise ValueError(
        f"期望 {NUM_CLASSES} 个类别，实际发现 {len(train_dataset.classes)} 个"
    )

train_loader = DataLoader(
    train_dataset,
    batch_size=BATCH_SIZE,
    shuffle=True,
    num_workers=0,
)
val_loader = DataLoader(
    val_dataset,
    batch_size=BATCH_SIZE,
    shuffle=False,
    num_workers=0,
)

print("类别到编号的映射：", train_dataset.class_to_idx)


class RoadDamageCNN(nn.Module):
    def __init__(self, num_classes: int) -> None:
        super().__init__()

        self.features = nn.Sequential(
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.BatchNorm2d(32),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(kernel_size=2),

            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(kernel_size=2),

            nn.Conv2d(64, 128, kernel_size=3, padding=1),
            nn.BatchNorm2d(128),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(kernel_size=2),
        )

        self.pool = nn.AdaptiveAvgPool2d((1, 1))
        self.classifier = nn.Linear(128, num_classes)

    def forward(self, images: torch.Tensor) -> torch.Tensor:
        features = self.features(images)
        pooled = self.pool(features)
        flattened = torch.flatten(pooled, start_dim=1)
        logits = self.classifier(flattened)
        return logits


def train_one_epoch(
    model: nn.Module,
    loader: DataLoader,
    loss_fn: nn.Module,
    optimizer: torch.optim.Optimizer,
) -> tuple[float, float]:
    model.train()
    total_loss = 0.0
    total_correct = 0
    total_samples = 0

    for images, labels in loader:
        images = images.to(device)
        labels = labels.to(device)

        optimizer.zero_grad()
        logits = model(images)
        loss = loss_fn(logits, labels)
        loss.backward()
        optimizer.step()

        batch_size = labels.size(0)
        total_loss += loss.item() * batch_size
        total_correct += (logits.argmax(dim=1) == labels).sum().item()
        total_samples += batch_size

    return (
        total_loss / total_samples,
        total_correct / total_samples,
    )


@torch.no_grad()
def evaluate(
    model: nn.Module,
    loader: DataLoader,
    loss_fn: nn.Module,
) -> tuple[float, float]:
    model.eval()
    total_loss = 0.0
    total_correct = 0
    total_samples = 0

    for images, labels in loader:
        images = images.to(device)
        labels = labels.to(device)

        logits = model(images)
        loss = loss_fn(logits, labels)

        batch_size = labels.size(0)
        total_loss += loss.item() * batch_size
        total_correct += (logits.argmax(dim=1) == labels).sum().item()
        total_samples += batch_size

    return (
        total_loss / total_samples,
        total_correct / total_samples,
    )


model = RoadDamageCNN(num_classes=NUM_CLASSES).to(device)
loss_fn = nn.CrossEntropyLoss()
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=LEARNING_RATE,
    weight_decay=1e-4,
)

best_val_accuracy = 0.0

for epoch in range(1, EPOCHS + 1):
    train_loss, train_accuracy = train_one_epoch(
        model,
        train_loader,
        loss_fn,
        optimizer,
    )
    val_loss, val_accuracy = evaluate(
        model,
        val_loader,
        loss_fn,
    )

    print(
        f"Epoch {epoch:02d} | "
        f"train_loss={train_loss:.4f} | "
        f"train_acc={train_accuracy:.3f} | "
        f"val_loss={val_loss:.4f} | "
        f"val_acc={val_accuracy:.3f}"
    )

    if val_accuracy > best_val_accuracy:
        best_val_accuracy = val_accuracy
        torch.save(
            {
                "model_state": model.state_dict(),
                "class_to_idx": train_dataset.class_to_idx,
            },
            "best_road_damage_cnn.pt",
        )

print(f"最佳验证集准确率：{best_val_accuracy:.3f}")
```

训练数据增强会随机裁剪、翻转并调整亮度和对比度，帮助模型适应摄像头视角、光照和取景范围的小幅变化喵。

验证数据不能使用随机增强，因为模型在不同验证轮次中应面对相同输入，才能得到可比较的性能指标喵。

三个卷积模块把通道数从 `3` 逐步增加到 `128`，同时通过最大池化降低空间尺寸，使网络逐层提取更抽象的特征喵。

`BatchNorm2d` 会根据批次统计量调整中间特征的分布，通常可以让训练更稳定，但在批量极小时需要特别留意统计量是否可靠喵。

`AdaptiveAvgPool2d((1, 1))` 会把每个通道的整张特征图汇总为一个数，因此分类器只需要接收 128 个特征，而不依赖固定的中间空间尺寸喵。

最后的线性层输出三个 logits，它们是未经概率归一化的类别分数，因此模型的 `forward` 方法中不需要手动调用 Softmax喵。

`CrossEntropyLoss` 会在内部完成适合损失计算的归一化，并要求真实标签是从 `0` 开始的整数类别编号喵。

训练阶段必须调用 `model.train()` 并执行反向传播，而验证阶段使用 `model.eval()` 和 `torch.no_grad()`，从而关闭训练专用行为并避免保存无用的梯度喵。

示例会保存验证集准确率最高的模型参数和类别映射，部署时必须保留这份映射，否则编号相同却语义不同会导致严重的类别解释错误喵。

这个网络适合帮助初学者理解完整流程，但真实公路项目通常还需要预训练骨干网络、更严格的数据划分、类别不平衡处理、误差分析和独立测试集喵。

## 常见误区

- **误区一：卷积核就是固定的边缘检测公式**——实际上首层卷积核可能学成类似边缘检测器的模式，但其具体权重由训练目标和数据共同决定喵。

- **误区二：网络越深就一定越准确**——实际上更深的网络更难优化，也更容易受到数据量、正则化、计算预算和架构设计的限制，残差连接只是让深层网络更容易训练喵。

- **误区三：池化只减少计算量而不会损失信息**——实际上池化会舍弃精确位置和细节，过早下采样可能直接抹掉细小裂缝喵。

- **误区四：验证集准确率很高就代表可以部署**——实际上同一路段的相似帧可能造成数据泄漏，而且类别不平衡、雨夜反光和新型路面材料都可能让部署表现明显下降喵。

- **误区五：分类模型能告诉我裂缝在哪里**——实际上分类模型主要判断整张图的类别，定位需要目标检测，像素级轮廓与面积测量则需要图像分割喵。

- **误区六：训练前应先对 logits 执行 Softmax 再传给交叉熵损失**——实际上 PyTorch 的 `CrossEntropyLoss` 直接接收 logits，提前执行 Softmax 既没有必要，也可能让数值计算和梯度表现变差喵。

## 苏格拉底式检查

- 如果同一条裂缝从图像左侧移动到右侧，我们为什么不需要为右侧位置重新学习一套卷积核，而全连接网络又为什么很难天然获得这种能力喵？

- 如果连续使用多次步幅为 `2` 的下采样后，一条只有两三个像素宽的裂缝消失了，你会优先考虑减小下采样速度、提高输入分辨率，还是单纯增加分类器层数，以及为什么喵？

## 快速总结

- 图像是具有通道和空间结构的张量，而 CNN 利用局部连接与权值共享高效处理这种结构喵。

- 卷积负责提取局部模式，激活函数引入非线性，池化或步幅卷积负责下采样，多层组合形成从边缘到物体的层级特征喵。

- ResNet 的残差连接为信息和梯度提供捷径，使更深的卷积网络更容易优化喵。

- CNN 具有较强的局部性和平移等变归纳偏置，而 ViT 更依赖数据学习全局关系与位置结构喵。

- 公路图像分类可以判断正常、裂缝或坑槽，但病害定位和轮廓测量还需要继续学习目标检测与图像分割喵。
