## Positioning

这次深入的是 Transformer 自注意力的“引擎内部”，也就是 Query 和 Key 到底怎么算出那个总和为 1.0 的注意力分布，把你前两次手写的分数换成模型真实的计算流程喵。

## Analogy

可以把 QKV 想成一场“相亲匹配会”喵。

每个 token 进场时都带三样东西：一张“择偶要求卡”（Query）、一张“自我介绍卡”（Key）、还有一份“真正能拿出手的嫁妆”（Value）喵。

当“放到”这个 token 想更新自己时，它拿出自己的择偶要求卡，去和在场每个人的自我介绍卡比对契合度喵。

契合度越高的人，分到的关注就越多；最后“放到”把每个人的嫁妆按契合度比例打包带走，组成自己的新理解喵。

注意三张卡是同一个 token 用三套不同的“变换矩阵”生成的，所以“我想找什么”“我能被怎么找到”“我实际贡献什么”可以是三件不一样的事喵。

## Core Mechanism

整个自注意力可以拆成五步，每一步都对应一个具体的数学操作喵。

**第一步：生成 Q、K、V 三套向量。** 每个 token 的输入向量 x，分别乘上三个可学习矩阵 W_Q、W_K、W_V，得到 Query、Key、Value 三个向量喵。这三个矩阵就是模型真正“训练出来”的参数，而不是你之前手写的分数喵。

**第二步：算相似度（点积）。** 当前 token 的 Query 和每个 token 的 Key 做点积，点积越大代表两个向量方向越接近，也就是“择偶要求”和“自我介绍”越契合喵。这一步就是你之前手写的 raw_scores 真正的来源喵。

**第三步：缩放（除以 √d_k）。** 把所有点积分数除以 √d_k（d_k 是 Key 向量的维度）喵。原因是维度越高，点积的数值天然越大，不缩放的话送进 softmax 会让分布过于极端（几乎全押一个 token），梯度也会变得很小很难训练喵。除以 √d_k 能把分数拉回一个稳定范围喵。

**第四步：softmax 归一化。** 把缩放后的分数送进 softmax，变成总和为 1.0 的权重分布——这正是你 06-27 学的那一步，现在你知道它前面还有点积和缩放两步喵。

**第五步：对 Value 加权求和。** 用这组权重去给每个 token 的 Value 向量做加权平均，得到的结果就是当前 token 融合了上下文之后的新表示喵。

一句话串起来就是：`Attention(Q, K, V) = softmax(QKᵀ / √d_k) · V`，这就是 Transformer 论文里那个著名公式喵。

关键在于：**分数不是关注词本身，而是关注 Query 和 Key 这两个“投影后的向量”有多契合**，所以同一个词换了上下文、换了层、换了注意力头，它的 Q/K/V 都会不同，关注对象也就跟着变喵。

## Code Example

> 用 3 个 token、2 维向量手算一遍完整流程，让公式落地喵。

```python
import numpy as np

tokens = ["机器人", "方块", "桌子"]

# 假设这些是已经算好的 Q/K/V 向量（真实模型里由 x·W_Q、x·W_K、x·W_V 得到）喵。
# 我们站在 "放到" 这个 token 的视角，它的 Query 是：
q_放到 = np.array([1.0, 1.0])

# 每个 token 的 Key（自我介绍卡）喵：
K = np.array([
    [1.0, 0.0],   # 机器人
    [1.0, 1.0],   # 方块
    [0.0, 1.0],   # 桌子
])

# 每个 token 的 Value（实际贡献的信息）喵：
V = np.array([
    [1.0, 0.0],   # 机器人
    [0.0, 1.0],   # 方块
    [1.0, 1.0],   # 桌子
])

d_k = q_放到.shape[0]            # Key 维度 = 2

# 第二步：点积相似度
scores = K @ q_放到              # [1, 2, 1]

# 第三步：缩放
scaled = scores / np.sqrt(d_k)  # 除以 √2 ≈ 1.414

# 第四步：softmax 归一化
weights = np.exp(scaled) / np.sum(np.exp(scaled))

# 第五步：对 Value 加权求和
context = weights @ V

for token, w in zip(tokens, weights):
    print(f"{token}: 注意力 {w:.3f}")
print("放到的新表示:", np.round(context, 3))
```

运行后你会看到 `方块` 拿到约 0.503 的注意力，`机器人` 和 `桌子` 各约 0.248 喵。

为什么方块最高？因为它的 Key=[1,1] 和 Query=[1,1] 方向完全一致，点积最大，契合度最高喵。

最后 `放到` 的新表示约为 `[0.496, 0.751]`，它不是某个原始词，而是三个 Value 按注意力比例混合出来的“融合理解”喵。

对比你 06-27 的代码：那次你直接手写 raw_scores 再做 softmax；这次的 `scores = K @ q` 就是那些 raw_scores 真正的产地，而 `/√d_k` 是你之前完全没接触过的缩放步骤喵。

## Common Misconceptions

第一个误区是以为 Q、K、V 是三个不同的 token喵。其实它们是同一个 token 经过三套不同矩阵投影出来的三个向量喵。

第二个误区是以为点积分数就能直接当权重喵。点积只是“原始契合度”，必须再经过缩放和 softmax 才变成总和为 1.0 的注意力分布喵。

第三个误区是搞不清为什么要除以 √d_k喵。它纯粹是数值稳定性的考虑：防止高维下点积过大导致 softmax 饱和、梯度消失，和“语义”无关喵。

第四个误区是以为 Query 和 Key 可以混用喵。如果当前 token 用自己的 Key（而不是 Query）去匹配，含义就变了——Query 代表“我要找什么”，Key 代表“我能被怎么找到”，角色不能互换喵。

## Socratic Check

先想一个问题：如果把上面例子里 `方块` 的 Key 改成 [0, 1]，它和 Query=[1,1] 的点积会从 2 降到 1，那它最终的注意力权重会变高还是变低喵？

答案是变低，因为点积变小意味着契合度下降，softmax 后分到的权重也会减少，关注就转移到其他 token 上了喵。

再想一个问题：为什么模型要单独学一个 W_V 来生成 Value，而不是直接拿 Key 当 Value 用喵？

答案是，“怎么被检索到”（Key）和“被选中后实际贡献什么内容”（Value）是两件事——一个 token 可能因为某个特征被关注，但它真正传递给下游的信息是另一套，分开两个矩阵能让模型更灵活地解耦这两种角色喵。

## Quick Summary

- 自注意力的完整公式是 `softmax(QKᵀ / √d_k) · V`，分五步：投影出 QKV → 点积 → 缩放 → softmax → 对 V 加权求和喵。
- 你之前手写的注意力分数，真正来源是 Query 和 Key 的点积；softmax 只是最后一步，前面还有点积和 √d_k 缩放喵。
- Q/K/V 是同一 token 用三套可学习矩阵投影出的三个向量，分别代表“我要找什么 / 我怎么被找到 / 我贡献什么”，这三个矩阵才是模型真正训练的参数喵。
