## Positioning

卡尔曼滤波是组合定位（GNSS/IMU/轮速融合）内部真正干活的状态估计引擎，也是多目标跟踪、AMCL 等一大类「预测—校正」算法的数学原型；前两次会话讲了它的直觉，这次把完整骨架补齐喵。

## Analogy

想象两个朋友分别告诉你现在几点：一个戴着走时不太准的机械表，一个刚瞄了一眼远处模糊的车站大钟喵。

你不会只信其中一个，也不会简单取平均，而是会想「机械表这几天大概偏了两分钟，大钟隔太远可能看错五分钟」，然后按各自的靠谱程度加权——越不靠谱的权重越低喵。

卡尔曼滤波做的就是这件事，只是它把「靠谱程度」精确地记成一个数（方差），并且每融合一次，都顺手更新自己对结果的把握有多大喵。

另一半类比是记账：滤波器的账本上永远有两栏，一栏写「我估计的状态是多少」，另一栏写「我对这个估计的不确定范围有多大」；预测让不确定范围变大，校正让它变小，账本永远两栏一起更新喵。

## Core Mechanism

### 1. 状态与协方差：滤波器随身带的两样东西喵

状态向量 x 是你想估计的量，例如组合定位里的位置和速度，写成 x = [p, v]喵。

协方差矩阵 P 描述估计的不确定度：对角线元素是各状态自身的方差（P00 是位置方差、P11 是速度方差），非对角元素 P01 记录「位置误差和速度误差是否联动」喵。

非对角元素非常关键：如果最近速度估高了，位置往往也会跟着偏多，这种关联就存在 P01 里，后面你会看到它让「只观测位置」也能修正速度喵。

卡尔曼滤波假设这些不确定度都长成高斯钟形，所以一个均值加一个协方差就能完整描述「我知道什么、不知道什么」喵。

### 2. 两个模型：运动怎么走、传感器怎么看喵

运动模型描述状态如何随时间演化：x_k = F·x_{k-1} + B·u + w，其中过程噪声 w ~ N(0, Q)喵。

以恒速模型为例，F = [ [1, dt], [0, 1] ]，意思是「新位置 = 旧位置 + 速度×dt，速度不变」；B·u 是已知的控制输入（例如 IMU 加速度），可有可无喵。

Q 是承认「模型是简化的」：真实车辆总有没建模的加速度、路面颠簸和参数误差，Q 越大表示你越不信自己的运动模型喵。

观测模型描述传感器读数和状态的关系：z = H·x + v，观测噪声 v ~ N(0, R)喵。

如果 GNSS 只测位置，H = [1, 0]，它把状态向量里「传感器能看到的那部分」挑出来；R 越大表示这个传感器越吵喵。

### 3. 预测步：账本外推，不确定度必然变大喵

预测就是让状态沿运动模型往前走一步喵：

```
x̂ = F·x̂                 （状态外推）
P  = F·P·Fᵀ + Q          （不确定度外推）
```

第二行有两层含义：F·P·Fᵀ 把旧的不确定度按运动关系传播（速度不确定会转化成位置不确定），末尾再加 Q，因为外推本身又引入了新的模型误差喵。

所以只预测不校正时，P 会一步步膨胀——这正是前面讲过的「隧道里 GNSS 消失后，位置还能输出，但把握越来越小」的数学形式喵。

### 4. 校正步：残差乘以增益，把估计拉向观测喵

观测到来后，执行四个动作喵：

```
y = z − H·x̂              （创新/残差：观测和预测差多少）
S = H·P·Hᵀ + R           （创新协方差：这个差值正常该有多大）
K = P·Hᵀ·S⁻¹             （卡尔曼增益：该信这个差值几成）
x̂ = x̂ + K·y             （按比例修正状态）
P  = (I − K·H)·P         （修正后不确定度收缩）
```

增益 K 本质是一个「信任分配比」：P 大（预测很没把握）而 R 小（传感器很准）时 K 接近 1，估计几乎跳到观测上；P 小而 R 大时 K 接近 0，观测几乎被无视喵。

每次校正后 P 收缩，表示「融合了新信息，我更有把握了」；下一轮预测 P 又变大，如此循环喵。

### 5. 一维推导：为什么说它是精度加权平均喵

把所有矩阵退化成一维（H=1），校正公式变成喵：

```
K  = P / (P + R)
x̂' = x̂ + K·(z − x̂) = (R·x̂ + P·z) / (P + R)
P' = (1 − K)·P = P·R / (P + R)
```

第二行就是初中加权平均：预测的权重是 R（对方的噪声），观测的权重是 P（自己的不确定度）——谁的方差小谁话语权大喵。

第三行更妙：P' = P·R/(P+R) 一定同时小于 P 和 R，也就是融合后的估计比两个来源单独哪一个都更确定，这就是「融合优于单传感器」的数学根据喵。

在线性高斯假设下可以证明，这个加权方式让均方误差最小，所以卡尔曼滤波不是拍脑袋的启发式，而是该假设下的最优估计器喵。

### 6. Q 和 R 从哪来、调错了会怎样喵

R 相对好定：来自传感器规格书、标定残差或静止数据统计，例如 GNSS 单点定位的位置标准差喵。

Q 更像工程判断：它描述「运动模型漏掉了什么」，例如恒速模型漏掉的加速度幅度；通常先按物理量级估一个，再用实际数据调喵。

Q 设得太小，滤波器会过度自信：P 收得很小、K 趋近 0，对真实机动反应迟钝，误差越攒越大甚至发散——症状是残差持续偏向一边喵。

Q 设得太大（或 R 太小），滤波器变得神经质：估计紧跟每一次观测噪声上蹿下跳，输出毛刺很多喵。

健康检查看创新序列：残差 y 应该是零均值的，且幅度与 S 预测的范围一致（比如 95% 的残差落在 ±2√S 内）；系统性偏离就说明 Q、R 或模型本身配错了喵。

### 7. 线性假设破掉之后：EKF、UKF 与 ESKF喵

标准卡尔曼滤波要求 F 和 H 是线性的，但真实车辆的姿态旋转、激光匹配观测都是非线性的喵。

EKF（扩展卡尔曼滤波）在当前估计点附近求雅可比矩阵做局部线性化，简单高效，是工程中最常用的一档，但强非线性或初值差时线性化误差会累积喵。

UKF（无迹卡尔曼滤波）不做线性化，而是选一组 sigma 采样点穿过非线性函数再重组均值和协方差，精度更好、代价稍高喵。

ESKF（误差状态卡尔曼滤波）是组合导航的主流选择：名义状态用 IMU 积分自由外推，滤波器只估计「名义轨迹偏了多少」这个小误差量；误差量始终很小、近似线性，对旋转和零偏的处理特别自然喵。

一句话选型：教科书系统用 KF，一般非线性用 EKF，非线性较强且算力允许用 UKF，带 IMU 的组合定位用 ESKF喵。

### 8. 工程要点：门限、可观性与数值细节喵

创新门限（gating）是对付野值的第一道闸：校正前先算 y²/S，超过 3σ 对应的门限（一维约 9）就拒绝这次观测，防止一次 GNSS 多路径跳变把整个状态带偏喵。

可观性指「观测到底能约束哪些状态」：只观测位置时，速度是靠 P 的非对角项被间接拉动的（增益里 K1 = P10/S 就是这条通道），若某状态与观测完全解耦，它的误差就只能靠模型硬扛喵。

数值上要保持 P 对称正定：长时间运行后浮点误差会让 P 失去对称性，工程实现常用 Joseph 形式的协方差更新或定期对称化来兜底喵。

多传感器频率不同也不要紧：预测步跟着最快的时钟跑，谁的观测到了就用谁的 H 和 R 做一次校正，这正是组合定位里 IMU 高频预测、GNSS 低频校正的运行方式喵。

## Code Example

下面的示例用「位置 + 速度」二维状态、恒速运动模型和只测位置的传感器，完整走一遍矩阵形式的预测—校正循环，并演示创新门限拒绝野值喵。

代码只依赖 Python 标准库，协方差按元素展开（P00、P01、P10、P11），每一行都能和上面的矩阵公式对上喵。

```python
import random

random.seed(42)

dt = 0.1
steps = 120

# 状态 x = [位置, 速度]，协方差 P 按元素展开保存喵
x_pos, x_vel = 0.0, 0.0
P00, P01, P10, P11 = 25.0, 0.0, 0.0, 4.0

# 过程噪声 Q 与观测噪声 R：Q 描述运动模型多不可信，R 描述观测多不可信喵
q_pos, q_vel = 0.001, 0.02
R = 1.0 ** 2

true_pos, true_vel = 0.0, 2.0
GATE = 9.0  # 3-sigma 门限：y^2 / S > 9 时拒绝该观测喵

rows = []

for i in range(steps):
    t = i * dt

    # 真实世界：恒速运动加一点随机加速度扰动喵
    accel = random.gauss(0.0, 0.1)
    true_pos += true_vel * dt + 0.5 * accel * dt * dt
    true_vel += accel * dt

    # ── 预测步：x = F x，P = F P Fᵀ + Q（F = [[1, dt], [0, 1]]）喵
    x_pos = x_pos + x_vel * dt
    n00 = P00 + dt * (P01 + P10) + dt * dt * P11 + q_pos
    n01 = P01 + dt * P11
    n10 = P10 + dt * P11
    n11 = P11 + q_vel
    P00, P01, P10, P11 = n00, n01, n10, n11

    # 传感器：位置观测，t=6.0 时人为注入一个 +8 m 的野值喵
    z = true_pos + random.gauss(0.0, 1.0)
    outlier = abs(t - 6.0) < 1e-9
    if outlier:
        z += 8.0

    # ── 校正步（H = [1, 0]）喵
    y = z - x_pos              # 创新（残差）喵
    S = P00 + R                # 创新协方差 S = H P Hᵀ + R 喵
    accepted = (y * y / S) <= GATE

    K0 = K1 = 0.0
    if accepted:
        K0 = P00 / S           # 卡尔曼增益 K = P Hᵀ S⁻¹ 喵
        K1 = P10 / S
        x_pos += K0 * y
        x_vel += K1 * y
        # P = (I - K H) P 喵
        n00 = P00 - K0 * P00
        n01 = P01 - K0 * P01
        n10 = P10 - K1 * P00
        n11 = P11 - K1 * P01
        P00, P01, P10, P11 = n00, n01, n10, n11

    rows.append((t, true_pos, x_pos, x_vel, P00, K0, outlier, accepted))

print(" time |  true |  est  | est_v |  P00  |  K0  | 备注")
for idx in (0, 1, 5, 20, 59, 60, 61, 119):
    t, tp, ep, ev, p00, k0, outlier, acc = rows[idx]
    tag = "野值被拒" if (outlier and not acc) else ("野值被吸收!" if outlier else "")
    print(f"{t:5.1f} | {tp:5.2f} | {ep:5.2f} | {ev:5.2f} | {p00:5.3f} | {k0:4.2f} | {tag}")
```

固定随机种子实际运行的输出如下喵：

```
 time |  true |  est  | est_v |  P00  |  K0  | 备注
  0.0 |  0.20 |  0.03 |  0.00 | 0.962 | 0.96 |
  0.1 |  0.40 |  0.57 |  0.22 | 0.501 | 0.50 |
  0.5 |  1.20 |  1.10 |  1.57 | 0.316 | 0.32 |
  2.0 |  4.21 |  3.54 |  1.60 | 0.189 | 0.19 |
  5.9 | 12.16 | 12.21 |  2.06 | 0.157 | 0.16 |
  6.0 | 12.36 | 12.42 |  2.06 | 0.187 | 0.00 | 野值被拒
  6.1 | 12.57 | 12.39 |  1.88 | 0.181 | 0.18 |
 11.9 | 25.01 | 24.99 |  1.90 | 0.157 | 0.16 |
```

先看增益的演化：初始 P00 = 25（承认自己完全不知道在哪），第一次校正 K0 = 0.96，估计几乎直接跳到观测上；随着 P 收缩，K0 逐步收敛到约 0.16 的稳态——滤波器从「全听传感器的」自然过渡到「主要靠模型、观测微调」喵。

再看速度：我们从未直接观测速度，但 est_v 从 0 爬到了真实值 2 附近，靠的正是预测步在 P01/P10 里积累的位置—速度关联，以及校正步里 K1 = P10/S 这条间接通道喵。

最后看野值：t = 6.0 的观测被加了 8 m，创新检验 y²/S 远超门限 9，该观测被整体拒绝（K0 = 0.00），估计纹丝不动；注意这一步 P00 反而从 0.157 涨到 0.187，因为预测步照常膨胀而校正被跳过——拒绝观测的代价是不确定度上升，这是诚实的喵。

## Common Misconceptions

- **误区 1：卡尔曼滤波就是一种滑动平均或低通滤波喵。**
  **纠正：** 平均的权重是固定的，而卡尔曼增益由 P 和 R 实时计算，还带着运动模型做外推；启动时 K≈1、稳态时 K≈0.16 的动态变化是任何固定权重平均做不到的喵。

- **误区 2：协方差 P 就是估计的真实误差喵。**
  **纠正：** P 只是滤波器「自己声称」的不确定度，Q、R 配错时它可以自信地错着——所以工程上要用创新一致性检验去核对 P 是否名副其实喵。

- **误区 3：噪声不是高斯分布，卡尔曼滤波就不能用喵。**
  **纠正：** 非高斯时它不再是全局最优，但仍是最优的「线性」无偏估计器，实践中依然大量使用；只有噪声重尾、多峰很严重时才需要换粒子滤波等工具喵。

- **误区 4：增益收敛到一个几乎不变的常数，说明滤波器出问题了喵。**
  **纠正：** 定常系统里 K 收敛到稳态是理论必然，一些嵌入式实现甚至直接预计算稳态增益来省算力；真正的异常信号是残差持续偏向一边，而不是增益稳定喵。

- **误区 5：遇到非线性，换成 EKF 就万事大吉喵。**
  **纠正：** EKF 只在估计点附近局部线性化，强非线性、大初值误差或糟糕的雅可比都会让它发散，该上 UKF、ESKF 或优化方法时不要硬撑喵。

## Quick Summary

- 卡尔曼滤波随身带两样东西：状态估计 x̂ 和不确定度 P，预测让 P 膨胀、校正让 P 收缩，循环往复喵。

- 增益 K = P·Hᵀ·(H·P·Hᵀ+R)⁻¹ 是信任分配比，一维时就是 P/(P+R) 的精度加权平均，融合后方差比任一来源都小喵。

- Q 表达对运动模型的不信任，R 表达对传感器的不信任；Q 太小滤波器睡着，Q 太大输出神经质，健康与否看创新序列喵。

- 非线性场景按强度选 EKF、UKF 或 ESKF，带 IMU 的组合定位主流是 ESKF喵。

- 工程三件套：创新门限拒野值、注意可观性（未观测状态靠协方差关联间接修正）、保持 P 对称正定喵。
