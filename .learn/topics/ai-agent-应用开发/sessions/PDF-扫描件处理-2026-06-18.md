## 定位

挂在 **RAG 知识库 → 文档解析与分块 → 文档解析（Parsing）** 之下。上一节说「解析就是把任意格式变成干净纯文本」，而 PDF 是所有格式里最难啃的一种，值得单拎出来讲。

## 类比

把 PDF 想象成「两种完全不同的东西，却穿着同一件外套」：

- **文字版 PDF（数字原生）**：像可复制粘贴的 Word 导出件，文字是「活」的，背后真存着字符。抽取它就像从盒子里直接拿字。
- **扫描版 PDF（图片版）**：本质是一沓拍下来的照片装进 PDF 外壳，文字是「死」的像素，复制不出东西，因为里面没有字符只有图。

处理 PDF 第一步永远是先问：**「这是文字版还是图片版？」** —— 决定走哪条路。

## 核心机制

### （1）先判断：文字版 vs 扫描版

朴素方法：试抽文字，抽出来基本为空就八成是扫描件。

```
PDF 进来
  ├─ 能抽到文字？ ─是→ 文字版：直接用文本抽取库
  └─ 抽出来是空的 ─→ 扫描版：先 OCR（图→字），再当文本处理
```

### （2）文字版 PDF：用抽取库

- **`pdfplumber`** —— 擅长表格和带坐标的精细抽取，适合复杂排版。
- **`PyMuPDF`（import 名 `fitz`）** —— 速度极快，抽文本/图片都行，工程最常用。
- **`pypdf`** —— 轻量，简单场景够用。

难点不在抽不出，而在抽出来很脏（分栏串行、页眉页脚混入、表格塌行），文字版主要工作是**清洗**。

### （3）扫描版 PDF：必须先 OCR

OCR（Optical Character Recognition，光学字符识别）= 看图认字，把像素识别成文字。

```
扫描 PDF → 每页转图片 → OCR 引擎逐页识别 → 拼回纯文本
```

主流引擎：
- **Tesseract**（开源老牌，`pytesseract`）—— 免费，中文需语言包，清晰文档够用。
- **PaddleOCR**（百度开源）—— 中文效果好，能识别表格/版面，国内首选。
- **云服务 OCR**（阿里/腾讯/AWS Textract）—— 准、省心，但花钱且数据出门，注意隐私。

两个现实坑：**慢**（逐页跑模型）和**不完美**（模糊/手写/印章易错），常做图像预处理（去噪、二值化、摆正）提准。

## 代码示例

```python
import fitz  # PyMuPDF，pip install pymupdf

def parse_pdf(path: str) -> str:
    doc = fitz.open(path)
    full_text = []

    for page_num, page in enumerate(doc):
        # ① 先尝试直接抽文字（文字版这条路）
        text = page.get_text().strip()

        if text:
            # 抽到了 → 文字版，直接用
            full_text.append(text)
        else:
            # ② 抽不到 → 八成是扫描页，转图片走 OCR
            pix = page.get_pixmap(dpi=300)        # 渲染成高清图，dpi 越高 OCR 越准
            img_bytes = pix.tobytes("png")
            text = ocr_image(img_bytes)            # 交给 OCR 引擎
            full_text.append(text)

        print(f"第 {page_num + 1} 页：抽到 {len(text)} 字")

    return "\n\n".join(full_text)


def ocr_image(img_bytes: bytes) -> str:
    from paddleocr import PaddleOCR
    import numpy as np, cv2
    ocr = PaddleOCR(use_angle_cls=True, lang="ch")   # 中文模型 + 自动摆正
    img = cv2.imdecode(np.frombuffer(img_bytes, np.uint8), cv2.IMREAD_COLOR)
    result = ocr.ocr(img)
    return "\n".join(line[1][0] for line in result[0]) if result and result[0] else ""
```

**逐段讲解：**
- `page.get_text()` 是分水岭：抽到非空走文字版、抽到空走 OCR，即判断图的代码版。
- `get_pixmap(dpi=300)` 把页面渲染成图片，`dpi` 是清晰度旋钮：太低认不清、太高又慢又占内存，300 是常用折中。
- OCR 段用 PaddleOCR，`lang="ch"` 中文模型，`use_angle_cls=True` 自动摆正歪斜文字。
- 关键心法：解析得到的 `full_text` 正是上一节「分块」的输入，两节由此接上。

## 常见误区

1. **❌「PDF 都一样，抽就完事」** —— 不判断文字版/扫描版就硬抽，扫描件得到空白还以为没问题。
2. **❌「OCR 出来的文字可以直接信」** —— OCR 有错字/漏字/串行，低置信度处要校验或抽检。
3. **❌「dpi 越高越好」** —— 提准有上限，过头只换爆炸的内存和耗时。
4. **❌「忽略表格和分栏」** —— 双栏直接 `get_text()` 常串成乱序，表格塌成一行；复杂排版用 `pdfplumber` 按坐标或上版面分析。
5. **❌「先分块再判断质量」** —— 解析是 RAG 天花板，脏文本喂下去后面救不回来，清洗要在分块前做。

## 苏格拉底式检查

> 一批混着文字版和扫描版的 PDF（事先不知哪页哪种）。若为省事对每页都无脑 OCR 会怎样？反之对每页都只 `get_text()` 不 OCR 又丢什么？

参考思路：全程 OCR → 文字版被拖去跑昂贵且可能出错的模型，又慢又引入错误；全程只抽文字 → 扫描页变空白、内容丢失。所以「先试抽、空了才 OCR」的**按页分流**才兼顾速度与完整性。

---

## 速记摘要

- PDF 分两类：文字版（字符是活的，直接抽）vs 扫描版（本质是图片，必须先 OCR）。
- 工程标准做法：先 `get_text()` 试抽，抽到空才转图片走 OCR —— 按页分流。
- 文字版工具：PyMuPDF（快）、pdfplumber（表格/坐标）、pypdf（轻量）；难点是清洗。
- 扫描版 OCR 引擎：PaddleOCR（中文首选）、Tesseract（开源）、云 OCR（准但花钱+隐私）。
- 表格、双栏排版要特殊处理；解析质量是 RAG 天花板，清洗要在分块前完成。
