# Learn Anything 命令教学喵

这份文档教你怎么用「Learn Anything」的 5 个斜杠命令，把 Claude 变成你的自适应私教喵。

> 提示：所有命令都是用自然语言驱动的，背后没有可运行的程序，真正干活的是 `.claude/skills/` 里的提示词喵。

---

## 一分钟总览喵

| 命令 | 干什么 | 什么时候用 |
|------|--------|-----------|
| `/learn <主题>` | 建立 / 加载一个学习主题，生成知识地图 | **第一步**，开一个新主题时 |
| `/learn-explain <概念>` | 递归式深入讲解某个概念 | 想把某个点彻底搞懂时 |
| `/learn-practice <概念>` | 出练习（写真实代码 or 对话讨论） | 学完概念想动手巩固时 |
| `/learn-review [主题]` | 用间隔重复分析进度，推荐下一步 | 不知道接下来学什么时 |
| `/learn-status [主题]` | 只读地可视化掌握度热力图 | 想快速看整体进度时 |

---

## 1. `/learn <主题>` —— 入口喵

**作用**：为一个想学的主题建立学习档案，或加载已有主题汇报进度喵。

- **新主题**：Claude 会创建 `.learn/topics/<主题>/` 目录，生成 `knowledge-map.md`（层级化的概念树）和 `state.yaml`（进度档案），然后带你看知识地图、引导你选起点喵。
- **已有主题**：Claude 读取已有数据、计算进度，给出个性化建议喵。

**示例喵：**
```
/learn Rust 所有权
/learn 傅里叶变换
/learn React Hooks
```

这是整个流程的第一步，其余命令都依赖它建立的数据喵。

---

## 2. `/learn-explain <概念>` —— 深入讲解喵

**作用**：把某个概念彻底讲透，采用固定的六段式结构喵：

> 定位 → 类比 → 核心机制 → 代码示例 → 常见误区 → 苏格拉底式反问

它会根据你在 `state.yaml` 里的信心值，自动判断你是初学 / 中级 / 高级，并调整讲解深度喵。讲完后还会把概念里可继续深挖的子点列出来，作为递归学习的入口喵。

**关键特性**：讲解内容会**在同一回合**被完整保存到 `sessions/<概念名>-日期.md`，并同步更新 `state.yaml`，所以你脱离聊天也能当笔记重读喵。

**示例喵：**
```
/learn-explain 借用检查器
/learn-explain useEffect 的依赖数组
```

---

## 3. `/learn-practice <概念>` —— 动手练习喵

**作用**：出练习帮你巩固，分两种模式喵：

- **项目模式（编程类主题）**：在 `.learn/topics/<主题>/exercises/<slug>/` 下写出**真实的练习文件**（README.md + starter 文件），你在 IDE 里打开做，做完 Claude 读你的代码、可能运行它、再给结构化反馈喵。
- **对话模式（概念类主题）**：直接在聊天里出题、给提示、批改喵。

难度会依据 `state.yaml` 自适应（初级 / 中级 / 挑战）喵。同样地，反馈和会话记录会在同一回合写盘并更新进度喵。

**示例喵：**
```
/learn-practice 实现一个链表
/learn-practice 解释 CAP 定理
```

---

## 4. `/learn-review [主题]` —— 复习规划喵

**作用**：分析你的学习数据，告诉你接下来该学 / 该复习什么喵。它用的是**间隔重复**优先级公式喵：

```
priority = (1 - confidence) * (距上次练习的天数 + 1) * w
w = 1.0 需练习 | 0.6 进行中 | 0.3 已掌握 | 0.1 未探索
```

也就是说：**信心越低、隔得越久、越需要练习**的概念会被优先推上来喵。它会给出四类建议：强化薄弱点 → 继续推进 → 探索新领域 → 间隔复习喵。

**示例喵：**
```
/learn-review              # 不带参数 = 跨所有主题总览
/learn-review Rust 所有权   # 只看某个主题
```

---

## 5. `/learn-status [主题]` —— 进度可视化喵

**作用**：一个**只读**命令，把你对各概念的掌握度画成热力图喵。它沿着知识地图的结构，给每个概念标上状态图标、练习次数、信心值，再附一个汇总面板（掌握统计、上次练习时间、已学天数）喵。

它不会改任何数据，纯看进度用喵。

**示例喵：**
```
/learn-status
/learn-status 傅里叶变换
```

## 推荐学习闭环喵

```
/learn <主题>          # ① 开主题，建知识地图
      │
      ▼
/learn-explain <概念>   # ② 挑一个概念讲透
      │
      ▼
/learn-practice <概念>  # ③ 动手练习巩固
      │
      ▼
/learn-review          # ④ 让它推荐下一步
      │
      ▼
/learn-status          # ⑤ 随时看整体进度
      └──── 回到 ② 循环 ────┘
```

## 背后的数据是怎么存的喵

所有命令共享同一份 per-topic 数据存储，这是整个系统的核心契约喵：

```
.learn/topics/<主题>/
├── knowledge-map.md   # 层级化概念树（领域 / 概念 / 细节）
├── state.yaml         # 每个概念的进度，唯一真相来源
├── sessions/          # 完整保存的讲解与练习记录
└── exercises/<slug>/  # 项目模式的练习文件
```

`state.yaml` 里每个概念都有 5 个状态，构成一条生命周期喵：

```
unexplored（未探索）→ in_progress（进行中）→ needs_practice（需练习）⇄ mastered（已掌握）
```

你不用手动改这些文件 —— 命令会在你学习的过程中自动维护它们喵。
